# 你好，我是 Dong

**中文** · [English](README.md)

我是一名从 HR 转向 AI 产品构建的人。我不是传统意义上的工程师，也不把自己定义成手写代码的开发者。我的工作是发现真实工作流里的问题，提出产品想法，梳理使用场景，测试结果，并把 AI 编程模型当作实现伙伴。

对我来说，vibe coding 不是省略产品思考，而是把清晰判断、实际需求和快速迭代变成真正可用的小工具。

## 我在做什么

我关注轻量、本地优先、服务于日常文件和知识工作流的小工具：

- 文件转换工具
- Markdown 与 Obsidian 工作流
- CSV、JSON 和文本清洗
- 多语言桌面端或浏览器工具
- 给非工程师使用的实用软件

## 重点项目

| 项目 | 说明 |
| --- | --- |
| [2markdown](https://github.com/mococouo/2markdown) | 批量把常见文件转换成适合 Obsidian 使用的 Markdown 笔记。 |
| [2json](https://github.com/mococouo/2json) | 在浏览器本地把 CSV 转成 JSON，支持多语言界面和编码处理。 |
| [markdown2txt](https://github.com/mococouo/markdown2txt) | 离线把 Markdown 转成 TXT，支持批量处理和多语言界面。 |

## 我的工作方式

- 从真实工作流里的摩擦出发，而不是从抽象功能清单出发
- 把模糊想法整理成清晰的产品提示词、用户流程和验收标准
- 使用 AI 编程模型完成构建、调试、打包和改进
- 在真实场景里试用工具，判断它是否真的好用
- 保持工具足够简单，让只想完成任务的人也能直接上手

## 当前兴趣

Vibe coding、AI 辅助产品构建、本地优先工具、Markdown 工作流、文件转换、数据清洗、多语言界面，以及帮助非工程师完成实际工作的软件。
